# RingMemoryApp 项目现有功能总结

## 1. 项目定位

`RingMemoryApp` 是一个基于 React Native 的智能戒指配套应用，当前主要围绕以下三条能力建设：

- 戒指设备扫描、连接与状态查看
- 戒指音频录制、播放与本地保存
- 录音片段上传与“记忆召回”交互雏形

当前 App 采用底部三 Tab 结构：

- `Memory`：记忆召回页
- `Devices`：设备与录音页
- `Settings`：设置页

主入口见 [App.tsx](/Users/ivy/Desktop/program/ChipletRing-APPSDK/RingMemoryApp/App.tsx)。

### 3.2 录音能力

当前已实现的录音模式：

- `ADPCM` 录音
- `PCM` 录音

当前录音流程：

1. 前端点击开始录音
2. 调用原生模块启动对应音频格式
3. 通过 BCLSDK 收取音频数据包
4. 在原生侧拼接、处理后转成 `wav`
5. 保存到本地目录
6. 回传 `onAudioSegmentReady` 给前端

录音文件保存路径：

- `Documents/ringmemoryapp/audio`

支持能力：

- 查询本地已保存录音
- 停止录音时落盘
- 录音过程中按片段保存

### 3.3 音频播放能力

当前已实现：

- 播放本地 `wav` 文件
- 停止播放
- 播放时展示进度

播放信息包含：

- 当前播放文件
- 当前播放时长
- 总时长
- 进度条可视化

## 4. 录音上传与服务端联动

当前已实现一条基础上传链路：

- 本地录音生成后，`Devices` 页会尝试自动触发上传
- 上传前会从 `AsyncStorage` 读取 token
- 若无 token，则跳过上传
- 上传接口为 `POST /api/app/uploadHistory`

相关文件：

- [useMemoryRecall.ts](/Users/ivy/Desktop/program/ChipletRing-APPSDK/RingMemoryApp/src/hooks/useMemoryRecall.ts)
- [api.ts](/Users/ivy/Desktop/program/ChipletRing-APPSDK/RingMemoryApp/src/services/api.ts)
- [storage.ts](/Users/ivy/Desktop/program/ChipletRing-APPSDK/RingMemoryApp/src/services/storage.ts)

当前还支持：

- 创建 token：`createToken`
- 获取最新历史：`getLatestHistory`

但这些接口在 UI 上还没有完整接通。

当前原生能力重点集中在 iOS：

- 已集成 `BCLRingSDK.xcframework`

## 7. 当前已知限制与未完成项

### 7.1 已有但仍在调试中的部分

- PCM 录音前段稳定性仍在持续调试
- 录音流异常恢复策略还在迭代
- 设备连接状态与录音流状态的边界处理仍有优化空间

### 7.2 仍是占位或未闭环的部分

- Memory 页 AI 召回仍为 mock
- Memory 卡片的播放/原文/分享未接真实能力
- Settings 页多数设置未落到真实配置存储或后端
- 上传记录、缓存管理、SDK 文档入口仍是 UI 占位

## 8. 一句话总结

这个项目当前已经完成了一个可跑通的“智能戒指录音 MVP”：

- 能扫设备、连设备
- 能录音、存本地、回放
- 能区分 ADPCM / PCM 两种录音模式
- 能把录音片段自动尝试上传

但“AI 记忆召回闭环”和“设置页真实业务能力”还处在雏形阶段，PCM 音频稳定性也仍在优化中。
